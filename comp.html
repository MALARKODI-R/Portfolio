RFID-Based Access Control System

This computational hardware project is a real-time access control system using the ESP32 microcontroller. The system uses an RFID card for user identification and grants or denies access based on UID matching. It features a Real-Time Clock (RTC) module for timestamping, a servo motor to open/close the gate, an LCD to display status, and logs the access attempt in Google Sheets over Wi-Fi.

Key Features:

✔ RFID Card Scanning
✔ Real-Time Clock Integration
✔ Servo-Controlled Gate
✔ Google Sheet Logging via Wi-Fi
✔ LCD Feedback + Buzzer + LED Alerts

Hardware Setup

Components include: ESP32-WROOM, MFRC522 RFID reader, DS1302 RTC module, SG90 Servo motor, LCD 16x2 with I2C, Buzzer, LED, and external 5V power for servo. All modules were interfaced using jumper wires on a breadboard. A CR2032 cell was used for RTC backup.

Interface Pins:

RFID → GPIO 5/18/23/19/4
RTC → GPIO 14/27/26
Servo → GPIO 13
LCD (I2C) → SDA 21, SCL 22
Buzzer → GPIO 15, LED → GPIO 2

System Workflow

1. User scans an RFID tag.
2. UID is verified against known entries.
3. Access is granted (servo opens gate) or denied.
4. Time is read from the RTC module.
5. UID, status, and timestamp are sent to Google Sheets using HTTP.
6. LCD displays the result, and buzzer/LED give audible/visual feedback.

Google Sheets Integration

The ESP32 sends HTTP GET requests to a Google Apps Script Web App, logging UID, status (Granted/Denied), and timestamp. The script writes these to a Google Sheet live, creating an easy-to-access logbook for access monitoring.

Example:

✅ UID: 0xED:0D:0C:32
📅 Date: 29/06/2025 11:21:30
📄 Status: Access Granted

System Output

The LCD clearly displays the scanned UID and whether access was granted or denied. The servo rotates to 180° upon successful access and returns to 0° after 3 seconds. Unauthorized cards trigger blinking LED and buzzer beeps.

Conclusion

This system effectively demonstrates computational hardware integration with cloud computing. It blends electronics, IoT, and web technologies to create a secure, automated, and loggable access control system—ideal for institutional gates, smart homes, or attendance monitoring systems.
